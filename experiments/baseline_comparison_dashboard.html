实验结论

无防护 Agent 的攻击误放行率最高，说明如果缺少工具调用前置安全治理，Agent 在面对恶意请求时容易直接执行危险操作。 简单关键词规则能够拦截部分显式风险，但仍存在明显攻击误放行问题。 本项目安全网关在同一评测集上实现了更低的攻击误放行率，并保持较低检测延迟，说明任务授权、风险评分和人工确认机制能够有效提升 Agent 工具调用安全性。
