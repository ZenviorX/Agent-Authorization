# Task19：安全评测体系 V2 与可复现实验框架建设

## 一、任务背景

随着项目逐步从基础原型发展为面向 AI Agent 工具调用的授权与安全防护系统，单纯依靠少量手动样例已经难以充分证明系统的有效性。为了进一步提升项目的工程完整性、实验可信度和国赛展示能力，本阶段围绕安全评测体系进行了系统化升级。

本任务的核心目标是：构建一套可扩展、可复现、可量化、可展示的 Agent 工具调用安全评测框架，使项目不只停留在“功能能运行”的层面，而是能够通过结构化实验数据证明系统在攻击拦截、风险确认、正常任务放行和运行时性能方面的综合能力。

## 二、主要工作内容

### 1. 升级评测脚本

本阶段对 gateway_eval_v2 评测脚本进行了升级，使其能够自动读取 security_cases 目录下所有符合 gateway_cases*.json 命名规则的样例文件。

相比原先固定读取少量样例文件的方式，新的自动加载机制具有更好的扩展性。后续只需要新增样例文件，即可自动纳入统一评测流程，不需要反复修改主评测脚本。

### 2. 扩展安全评测样例集

本阶段新增 gateway_cases_v3.json，并结合原有样例将评测集扩展到 70 条。

当前评测集覆盖 normal、attack、suspicious 三类样例，进一步包括正常工具调用、路径穿越、敏感文件访问、提示注入、敏感信息外发、高危命令执行、SQL 风险、未知工具调用、低置信度 Agent 计划等场景。

这些样例使项目评测不再局限于简单关键词检测，而是更接近真实 AI Agent 在工具调用过程中可能遇到的安全风险。

### 3. 完善评测指标

新的评测流程自动统计总体一致率、攻击阻断/升级确认率、攻击误放行率、正常任务误拒绝率、平均检测延迟、决策分布、样例文件分布、场景分类分布和分类准确率等指标。

这些指标同时关注安全性和可用性，避免只追求攻击拦截而忽略正常任务体验，也为后续作品报告中的实验分析提供了可量化依据。

### 4. 增加多格式实验输出

评测脚本运行后会自动生成 CSV、JSON 和 Markdown 三类结果文件。

其中，CSV 文件用于保存每条样例的详细评测记录；JSON 文件用于保存结构化统计摘要；Markdown 文件可直接作为作品报告中的实验结果片段。

这种多格式输出方式提升了实验结果的可复用性，也方便后续进行数据分析、报告撰写和答辩展示。

### 5. 增加 HTML 可视化仪表盘

本阶段新增 generate_eval_dashboard.py 脚本，可以根据评测结果自动生成 gateway_eval_dashboard.html 可视化仪表盘。

该仪表盘展示了总样例数、通过样例数、总体一致率、平均检测延迟、攻击阻断/确认率、攻击误放行率、正常任务误拒绝率、分类分布和样例明细等信息。

该页面可以直接用于作品报告截图、README 展示和答辩演示，增强项目成果的直观性。

### 6. 增加一键复现实验脚本

本阶段新增 scripts/run_full_gateway_eval.ps1，用于一键完成完整评测流程。

运行该脚本后，系统会自动执行安全评测、生成结构化结果、生成 HTML 仪表盘，并可选择自动打开可视化页面。

这一改进降低了实验复现成本，也体现了项目作为工程作品的规范性和完整性。

### 7. 增加评测方法说明文档

本阶段新增 docs/security_evaluation_v2.md，对评测目标、样例组织方式、字段设计、覆盖场景、评测指标、实验输出和后续扩展方向进行了系统说明。

该文档为后续作品报告和答辩讲解提供了理论和方法支撑，使评测体系不只是代码实现，而是形成了相对完整的方法论说明。

## 三、实验结果

当前评测体系共加载 70 条样例，评测脚本能够稳定生成 CSV、JSON、Markdown 和 HTML 四类结果文件。

在当前测试结果中，系统整体评测通过，攻击样例能够被拒绝或升级确认，正常样例没有出现明显误拒绝，平均检测延迟保持在较低水平，说明该安全网关具备作为 Agent 工具调用前置检查模块的可行性。

## 四、任务价值

本阶段工作显著提升了项目的实验可信度和展示完整性。

从功能角度看，项目已经能够对多类工具调用风险进行自动化批量评测；从实验角度看，项目已经具备可复现、可扩展、可量化的评测流程；从展示角度看，项目已经能够通过 HTML 仪表盘直观呈现安全防护效果；从参赛角度看，这一改进有助于将项目从普通功能型系统提升为具有实验支撑和研究表达的完整作品。

## 五、后续方向

后续可以继续将评测集扩展到 100 条以上，并增加多步攻击链专用评测、传统 RBAC 对比实验、关键词规则对比实验、无防护 Agent 对比实验和不同风险阈值下的消融实验。

同时，可以考虑将评测结果进一步接入前端页面，使系统不仅能展示单次工具调用的安全决策，也能展示整体安全评测结果和攻击链防护效果。
